In a **legacy healthcare reporting system** with limited documentation, a daily report running on a **1M+ record database** took more than **24 hours** to complete. After tracing the data flow, I realized the bottleneck was not the SQL itself, but **redundant joins**, **repeated aggregations**, and **duplicated processing across system layers**. By redesigning how data was accessed and processed, I reduced the report runtime from **over 24 hours to under 1 hour**. This experience reinforced an important lesson: **performance issues are often system design problems, not just query problems**.
